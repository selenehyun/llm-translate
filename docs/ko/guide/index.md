# llm-translate란?

llm-translate는 대규모 언어 모델을 사용하여 문서를 번역하는 CLI 도구입니다. 일관성, 정확성 및 형식 보존이 중요한 기술 문서를 위해 특별히 설계되었습니다.

## 주요 기능

### 용어집 적용

도메인 특화 용어를 한 번 정의하고 모든 문서에서 일관되게 번역되도록 보장합니다.

```json
{
  "terms": [
    {
      "source": "API endpoint",
      "targets": { "ko": "API 엔드포인트", "ja": "APIエンドポイント" }
    },
    {
      "source": "Kubernetes",
      "doNotTranslate": true
    }
  ]
}
```

### Self-Refine 품질 제어

llm-translate는 단순히 한 번 번역하고 끝내지 않습니다. Self-Refine 반복 개선 프로세스를 사용합니다:

1. **초기 번역** - 용어집을 사용하여 첫 번째 번역 생성
2. **품질 평가** - 의미론적 정확성, 유창성, 용어집 준수 및 형식 보존에 대해 평가 점수 부여
3. **검토** - 품질 임계값을 충족하지 못한 경우 특정 문제 식별
4. **개선** - 특정 수정 사항 적용
5. **반복** - 품질 >= 임계값에 도달하거나 최대 반복 횟수에 도달할 때까지 계속

### 프롬프트 캐싱

Claude 모델의 경우 llm-translate는 자동으로 프롬프트 캐싱을 사용하여 비용을 절감합니다:

- 시스템 지침 및 용어집은 청크 전체에서 캐시됩니다
- 후속 요청은 캐시된 토큰을 90% 비용 절감으로 사용합니다
- 많은 청크가 있는 대용량 문서에 특히 효과적입니다

### 형식 보존

이 도구는 AST 기반 파싱을 사용하여 다음을 보존합니다:

- Markdown 형식(헤더, 목록, 표, 코드 블록)
- HTML 태그 및 속성
- 링크 및 이미지
- 문서 구조 및 계층 구조

## 사용 사례

- **기술 문서** - README, API 문서, 사용자 가이드 번역
- **지식 기반** - 다국어 지원 기술 자료
- **제품 콘텐츠** - 릴리스 노트, 변경 로그, 기능 설명
- **개발자 리소스** - 튜토리얼, 가이드, 코드 주석
